# Deployment Guide: Cloudflare Pages

This project is designed for static hosting with free-tier friendly infrastructure.

## Build artifacts

- App shell: `dist/`
- Versioned data shards: `public/data/` generated by `bun run build:data`

Use this build flow before any deploy:

```bash
bun install
bun run build:data
bun run build
```

## Runtime env

- `VITE_DATA_BASE_URL`: optional external CDN/data origin.
  - Example jsDelivr branch URL: `https://cdn.jsdelivr.net/gh/<org>/<repo>@data-cdn/data`
  - If this URL fails, runtime falls back to same-origin `/data`.

## Cloudflare Pages

Use these build settings in Cloudflare Pages:

- Build command: `bun run build:data && bun run build`
- Build output directory: `dist`
- Node/Bun runtime: Bun 1.x

If data remains same-origin, copy `public/data` into deployed static output as part of build (Vite handles this automatically).

## Notes

- Keep immutable version folders (`vYYYYMMDD`) in data output.
- Never mutate existing versioned shard files; publish a new version instead.
- Prefer short app-shell cache and long immutable data cache.
